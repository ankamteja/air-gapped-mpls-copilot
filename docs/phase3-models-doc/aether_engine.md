# Phase 3 — Predictive Engine & Corroboration Framework

**Goal:** Train ML models on labeled telemetry to detect, classify, and time faults before SLA breach. Corroborate ML predictions with a deterministic graph model before any autonomous action is taken.

**Status:** ✅ Done — all 3 LSTM models trained on `dataset_large.csv` (100k rows, 6 classes). Classifier val accuracy: **91.73%**. Regressor RMSE: **11.41s**. All models Ed25519-signed. Graph clonal search verified. Full corroboration pipeline demonstrated. Taxonomy single-source-of-truth enforced. v4 adds: EMA threshold, digital twin, attention heatmap, syslog parser, model signing, operator feedback CLI, benchmark harness.

---

## Design Goals

1. **Unsupervised Anomaly Detection:** LSTM Autoencoder learns healthy baseline; high reconstruction loss = anomaly.
2. **Supervised Fault Classification:** BiLSTM + Attention Classifier identifies the specific fault class (6 classes).
3. **Dual-Model Corroboration Gate:** A stochastic neural network never directly touches production configs; it must agree with the deterministic graph model first.
4. **Edge Policy Engine (EPE) & Autonomy:** Consult an operator-defined Autonomy Policy Matrix and compile an Anomaly Context Packet (ACP) containing CLI configuration fixes.

## Technical Details

### 1. Clonal State-Space Search

Instead of fixed failover tables, [`graph_model.py`](../../phase3-models/graph_model.py) generates virtual copies ("clones") of the topology:

- `CLONE_BASELINE`: Standard routing, unmodified.
- `CLONE_REROUTED_OVERLAY`: Deprioritises the degraded link (cost → 999999), shifting traffic onto the SD-WAN backup tunnel (pe1↔pe2 direct, 40ms, 2Mbps).
- `CLONE_QOS_THROTTLED`: Primary path stays active, but non-critical DB bulk traffic is throttled from 4.5Mbps to 1.15Mbps, preserving VoIP headroom.

Each clone is evaluated against the live traffic matrix (VoIP + DB + SSH). The engine scores each clone on total path delay + capacity saturation penalties, selecting the minimum-score clone as the recommended action.

### 2. Dual-Model Safety Gate

[`aether_corroborator.py`](../../phase3-models/aether_corroborator.py) is the decision gateway:

```
+------------+       +-------------+
| LSTM (ML)  |       |   NetworkX  |
| Prediction |       | Projection  |
+------------+       +-------------+
      |                     |
      +----------+----------+
                 |
                 v
      +---------------------+
      |  Corroboration Loop |
      +---------------------+
                 |
        Is there agreement?
       /                 \
     Yes                  No
     /                      \
+------------------+    +-------------------------+
| Consult Autonomy |    | SAFETY TRIP:            |
|  Policy Matrix   |    | Downgrade execution     |
+------------------+    | to RECOMMEND_ONLY       |
                        +-------------------------+
```

- **Agreement:** LSTM predicts congestion on pe1→p1 AND NetworkX graph projects link saturation → both agree → check autonomy policy.
- **Disagreement (Safety Trip):** LSTM predicts fault but graph shows the physical interface can carry the traffic with zero saturation (transient noise) → EPE trips to `RECOMMEND_ONLY`. No autonomous action.

### 3. Operator-Configurable Autonomy Matrix

Defined in [`taxonomy.py`](../../phase3-models/taxonomy.py) as `ACTION_POLICY`. Operators control per-action-class confidence thresholds and auto-execute permission:

```python
ACTION_POLICY = {
    "REROUTE_BRANCH":    {"min_conf": 0.80, "auto_execute": True},   # Auto-reroutes around branch failures
    "QOS_SHAPE_QUEUE":   {"min_conf": 0.75, "auto_execute": True},   # Auto-throttles non-critical queues
    "CORE_PATH_FAILOVER":{"min_conf": 0.90, "auto_execute": False},  # Recommend only — never auto on core
    "NODE_ISOLATION":    {"min_conf": 0.99, "auto_execute": False},  # Operator must confirm
}
```

Safety floors enforced in code regardless of this table: model disagreement always downgrades to `RECOMMEND_ONLY`; every auto-executed action is logged and reversible.

### 4. Anomaly Context Packet (ACP)

The structured handoff between models, graph engine, and the EPE. An ACP JSON contains:

1. **Severity Level** — escalated to `CRITICAL` for control-plane flaps or TTF < 30s.
2. **Telemetry Snapshot** — per-node interface counters at time of detection.
3. **ML Analysis** — anomaly flag, reconstruction loss, predicted fault class + confidence, estimated TTF.
4. **Graph Analysis** — saturated links list, impacted paths.
5. **Corroboration** — agreement flag, rationale string, recommended action, execution mode.
6. **Mitigation Commands** — exact `docker exec vtysh` / `tc` commands ready to paste or auto-execute.

### 5. v4 Additions

**EMA Self-Calibrating Threshold (`inference_engine.py`)**  
Replaces the fixed `3× mean` multiplier for anomaly detection. The `EMAThreshold` class maintains an exponential moving average of reconstruction loss (mean + k×std), adapting automatically to channel-specific noise. Includes a 50-sample warmup using the checkpoint's static threshold as fallback.

**Attention Heatmap (`predictive_engine.py`)**  
The `LSTMAttentionClassifier` now has a `FeatureHeatmap` linear head on the BiLSTM context vector → per-feature importance scores `[B, Features]`. The top-5 contributing features are attached to every ACP (`acp.top_features`) for operator explainability. This is read-only — it never feeds the corroboration gate.

**Digital Twin (`digital_twin.py`)**  
Uses Holt-Winters trend forecasting to project telemetry t+30s into the future, then runs both the actual and forecasted states through the graph model. Normalized divergence = `|forecast_score − actual_score| / max(actual_score, 0.001)`. A high divergence (>0.3) is appended to the ACP rationale as a Predictive Safety Factor (PSF) — it widens operator context but never blocks auto-execute.

**Per-Service SLA Scoring (`graph_model.py`, `config/sla_config.yaml`)**  
Each service class (voip, database, bulk_transfer, default) has configurable latency/loss thresholds and a priority weight. VoIP violations are penalized more heavily than bulk transfer in the graph's bottleneck score.

**Ed25519 Model Integrity (`model_integrity.py`)**  
All three model weights are signed with Ed25519 on the training box. The private key stays offline; only `keys/aether_model_key.pub.pem` is committed. The inference engine verifies all signatures at startup; a failed check triggers `IntegrityError` and drops the engine to SUPERVISED mode.

**FRR Syslog Parser (`phase2-telemetry/syslog_parser.py`)**  
Tails `/var/log/frr/frr.log` inside containers, extracting BGP and OSPF adjacency events via regex. Computes a rolling `routing_instability_score` (0–1) over the last 30 seconds. Can be wired into the inference pipeline as an additional feature channel.

**Benchmark Harness (`benchmark_harness.py`)**  
Replays `dataset.csv` at 1× speed, identifies fault onset timestamps, and measures detection lead time relative to simulated SLA breach. Result on dataset.csv: **9/9 scenarios detected, average 343 seconds before SLA breach**.

### 6. Fault Taxonomy

All class IDs, labels, display names, and action mappings live in a single file — [`taxonomy.py`](../../phase3-models/taxonomy.py). Every other file imports from it; no file redefines these mappings.

| ID | Label | Display Name | Action Class |
|---|---|---|---|
| 0 | `Healthy` | Healthy | NO_ACTION |
| 1 | `latency` | Latency Drift | REROUTE_BRANCH |

## 7. Synthetic Dataset Generator (`generate_dataset.py`)

Produces `dataset_large.csv` — a 100,000-row training dataset that exactly matches the 140-column schema output by `data_collector.py`, with no real containerlab instance required.

**Design goals:**

- **Column-exact:** Matches every column name and order from `data_collector.py` (timestamp, fault_label, fault_location, 7 nodes × interface counters + FRR state).
- **Monotonic counters:** Cumulative `rx_bytes`, `tx_bytes`, `rx_packets`, etc. only increase, mimicking real kernel interface counters.
- **Temporal episodes:** Each fault has gradual onset → sustained plateau → recovery using a `_ramp()` function, not instantaneous jumps.
- **Absolute fraction model:** Fault signals are specified as absolute packet-drop / error fractions (e.g., 3% loss → `rx_drop_frac=0.030`), not multipliers, ensuring ML-distinguishable per-class signals.
- **Diurnal variation:** Traffic has a sinusoidal business-hour factor (0.75 overnight → 1.30 peak) to prevent models from fitting on time-of-day rather than fault patterns.
- **FRR state propagation:** BGP/OSPF/LDP session state changes cascade correctly (e.g., p1 flap clears all PE sessions on both sides).

**Class distribution in `dataset_large.csv`:**

| Class | Rows | Fraction |
|---|---|---|
| Healthy | 40,000 | 40% |
| latency | 12,000 | 12% |
| loss | 12,000 | 12% |
| corrupt | 12,000 | 12% |
| rate | 12,000 | 12% |
| flap | 12,000 | 12% |
